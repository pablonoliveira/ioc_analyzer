Visão Geral

Plataforma para análise, gestão e correlação de Indicadores de Comprometimento (IoCs) com Vulnerabilidades (CVEs).
Desenvolvida para blue teams, analistas de Threat Intelligence e SOCs.

Instalação (CLI)

git clone https://github.com/pablonoliveira/ioc_analyzer.git
cd ioc_analyzer
python -m venv venv
# Ative o venv
pip install -r requirements.txt
cp .env.example .env
# Edite suas chaves/dependências no .env

Executando

python webapp.py
Dica: Acesse pelo browser em http://localhost:5000 (ou http://SEU_IP_LOCAL:5000 em rede LAN).

Funcionalidades Detalhadas

Dashboard

  • Gráficos interativos (por tipo, severidade, origem, etc.)
  • Estatísticas em tempo real

Painel de IoCs

  • Busca, filtro rápido, CRUD, prevenção de duplicidade
  • Paginação (10 por página)
  • Classificação por tipo e severidade
  • Exclusão individual (segura)

Painel de CVEs

  • Busca multi-fonte (NVD, CIRCL, CISA KEV), filtro e tradução automática
  • Salvar nos favoritos e prevenção de duplicidades
  • Paginação, filtros por severidade, ID, origem

Upload de Logs

  • Drag-and-drop de arquivos .log, .txt, .csv
  • Extração automática de IOCs

Configuração de Ambiente (.env)

Use o arquivo .env para configurar suas chaves de API de Threat Intelligence, porta do servidor, debug e integrações futuras: 


# APIs de Threat Intelligence
ABUSEIPDB_API_KEY=sua_chave_aqui
VIRUSTOTAL_API_KEY=sua_chave_aqui

# Configuração do Servidor Flask
FLASK_DEBUG=True
FLASK_HOST=0.0.0.0
FLASK_PORT=5000
Nota: O sistema roda normalmente sem as APIs, mas com funcionalidades limitadas para enriquecimento e reputação.

Segurança e Recomendações

  • Use sempre ambiente local/restrito
  • Configuração de firewall recomendada em ambientes produtivos
  • Debug deve ser desabilitado (debug=False) para produção
  • Use HTTPS e autenticação se for expor externamente

Roadmap (v2.2+)
